APLICACIÓN AL CASO SEMESTRAL
===================================================================
SmartLogix es una plataforma para PYMEs de eCommerce que reemplaza
sistemas monolíticos con una arquitectura de microservicios. El caso
semestral identifica tres problemas principales y esta solución los
aborda con patrones de diseño específicos:

PROBLEMA 1 — Sincronización deficiente de inventarios
  Solución: Microservicio Inventario con Repository Pattern
  → JpaRepository abstrae el acceso a PostgreSQL permitiendo
    sincronizar stock en tiempo real entre múltiples bodegas y
    pasillos sin acoplar la lógica de negocio a la base de datos.
  → Kafka publica eventos de cambio de stock para notificar a otros
    servicios de forma asíncrona y desacoplada.

PROBLEMA 2 — Errores en la asignación de pedidos
  Solución: Microservicio Orden con Factory Method (OrdenFactory)
  → OrdenFactory centraliza y estandariza la creación de órdenes,
    eliminando inconsistencias al instanciar pedidos desde distintos
    flujos (cliente, admin, transportista).
  → Repository Pattern en OrdenRepository garantiza trazabilidad
    completa del ciclo de vida del pedido en PostgreSQL via JPA.

PROBLEMA 3 — Demoras en coordinación con transportistas
  Solución: Gateway con Circuit Breaker (Resilience4j)
  → El Circuit Breaker en el Gateway detecta fallos en microservicios
    downstream (Orden, Inventario) y activa un fallback automático,
    evitando que un servicio caído bloquee toda la plataforma.
  → El BFF (Gateway) centraliza la comunicación entre el frontend
    Angular y los microservicios, reduciendo latencia y simplificando
    la integración con transportistas.

PATRÓN TRANSVERSAL — Escalabilidad y desacoplamiento
  → Cada microservicio es independiente (puerto propio, BD propia),
    permitiendo escalar Inventario u Orden ante picos de demanda sin
    afectar el resto del sistema.
  → Arquetipos Maven (Spring Boot) garantizan estructura uniforme en
    todos los servicios, facilitando mantenimiento y extensibilidad.

COMPONENTES BACKEND — Detalle
===================================================================

1. Gateway / BFF (Backend For Frontend)
-------------------------------------------------------------------
URL:  https://github.com/benjazzx/SmartLogix/tree/main/Gateway
Puerto: 8080
Patrón: API Gateway + Backend For Frontend (BFF)
Descripción:
  Punto de entrada único de la plataforma. Enruta todas las solicitudes
  del frontend hacia los microservicios correspondientes. Valida el token
  JWT en cada petición mediante un filtro global (JwtFilter). Implementa
  Circuit Breaker con Resilience4j para manejar fallos entre servicios.
  Incluye GlobalExceptionHandler para respuestas de error unificadas.

2. Microservicio Users
-------------------------------------------------------------------
URL:  https://github.com/benjazzx/SmartLogix/tree/main/Users
Puerto: 8082
Patrón: Repository Pattern + MVC + JWT Authentication
Descripción:
  Gestión de usuarios: registro, autenticación y generación de tokens JWT.
  Almacena contraseñas con BCrypt. Encripta el RUT para privacidad. Expone
  endpoints de login, registro, cambio de clave y perfil de usuario.
  Base de datos: PostgreSQL con JPA/Hibernate.

3. Microservicio Rol
-------------------------------------------------------------------
URL:  https://github.com/benjazzx/SmartLogix/tree/main/Rol
Puerto: 8081
Patrón: Repository Pattern + MVC
Descripción:
  Catálogo de roles del sistema (admin, cliente, transportista, bodeguero).
  Asigna roles automáticamente según el dominio del email durante el registro.
  Consultado por Users para determinar permisos en el JWT.

4. Microservicio Inventario
-------------------------------------------------------------------
URL:  https://github.com/benjazzx/SmartLogix/tree/main/Inventario
Puerto: 8083
Patrón: Repository Pattern + MVC + Mensajería asíncrona (Kafka)
Descripción:
  Gestión de stock en tiempo real. Organiza el inventario en Bodegas,
  Pasillos y Estantes. Publica eventos Kafka cuando el stock cambia.
  Conectado a PostgreSQL con JPA. Incluye pruebas unitarias con cobertura
  mínima del 60%.

5. Microservicio Orden
-------------------------------------------------------------------
URL:  https://github.com/benjazzx/SmartLogix/tree/main/Orden
Puerto: 8084
Patrón: Repository Pattern + Factory Method (OrdenFactory) + MVC
Descripción:
  Gestión del ciclo de vida de pedidos: creación, asignación a transportista,
  entrega y confirmación. Implementa OrdenFactory para centralizar la
  creación de órdenes. Mantiene historial completo de estados. Consume
  usuarios y productos vía RestTemplate (HTTP client). Pruebas unitarias
  con JUnit 5 y Mockito.

6. Microservicio Producto
-------------------------------------------------------------------
URL:  https://github.com/benjazzx/SmartLogix/tree/main/Producto
Puerto: 8085
Patrón: Repository Pattern + MVC
Descripción:
  Catálogo de productos con imágenes almacenadas en base de datos.
  Permite filtrar por categoría y país de origen. Expone endpoint GET
  público para consulta desde el frontend sin autenticación.

7. Microservicio Estado
-------------------------------------------------------------------
URL:  https://github.com/benjazzx/SmartLogix/tree/main/Estado
Puerto: 8086
Patrón: Repository Pattern + MVC
Descripción:
  Catálogo de estados de orden (Pendiente, En tránsito, Entregado, etc.).
  Asigna estados según el rol del usuario. Incluye DataInitializer para
  sembrar los estados base al iniciar la aplicación.

8. Microservicio Configuracion
-------------------------------------------------------------------
URL:  https://github.com/benjazzx/SmartLogix/tree/main/Configuracion
Puerto: 8087
Patrón: Repository Pattern + MVC + JWT Filter
Descripción:
  Gestión de preferencias de usuario por clave-valor (tema, idioma, etc.).
  Valida el JWT localmente mediante JwtFilter propio. Requiere autenticación
  para todas las operaciones. Tests unitarios para JwtUtil, JwtFilter y
  Controller (18 tests, cobertura >60%).

INSTRUCCIONES PARA LEVANTAR EL SISTEMA
===================================================================

Requisitos previos:
  - Docker Desktop instalado y corriendo
  - No se necesita PostgreSQL instalado (viene en el contenedor)

Levantar todo con Docker Compose:
  cd Smart/
  docker compose up -d
  → API disponible en http://localhost:8080
  → PostgreSQL disponible en localhost:5433 (usuario: postgres / postgres)
  → pgAdmin para ver BD: conectar a localhost:5433

Levantar todo incluyendo reconstrucción de imágenes locales:
  docker compose up -d --build

Levantar microservicio individual (desarrollo):
  cd Smart/<Microservicio>/
  mvn spring-boot:run

Levantar frontend:
  cd SmartLogix-main/SmartLogix-main/SmartLogix/
  npm install
  npm start
  → http://localhost:4200

Ejecutar tests y cobertura:
  cd Smart/<Microservicio>/
  mvn verify
  → Reporte en target/site/jacoco/index.html
